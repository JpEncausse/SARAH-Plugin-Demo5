Structure

La documentation d'un plugin est intégrée dans une page portail contruite avec Twitter Bootstrap.

  • Les titres commencent à H4
  • Le template est structuré en grille de 12 cases (span1 à span12)

Code

Il est possible de citer des termes techniques avec <code>.

  Et il est possible de mettre des exemples de code avec la balise <pre>

Portlet

Il est possible de surcharger les faces des portlets avec portlet.html et portlet_back.html. Si les 2 sont définis les boutons apparaissent dans le dos.

Taille

Il est possible de configurer les portlets pour qu'elles occupent plus de place. Et les préconfigurer dans le plugin.prop.

Code

Une portlet peut déclarer ses feuilles de style

<% stylesheet('assets/'+key+'/portlet.css') %>

Le singleton SARAH est accessible depuis le HTML. Il est donc possible d'accèder au plugin.

var plugin = SARAH.ConfigManager.getModule(key);

Le plugin peut exposer des fonctions qui seront ensuite disponibles depuis la portlet

var data;
exports.mafunc = function(){ return data; }

<%= plugin.mafunc(); %>

C'est une solution pour remonter de l'info des plugins. Et mettre en place un stream de donnée WebSocket.

Enfin les liens qui ont la classe ajax sont executé en ajax.

Les images sont stockées dans le dossier /images et accessible via l'URL /assets/{plugin}/images/{file.ext}
